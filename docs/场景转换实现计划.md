**文件：`client/src/game/scenes/MapScene.ts`**（核心）

这是最关键的文件。流程：
1. `init(data)` 记录mapId
2. `create()` → `loadMap(mapId)`
3. `loadMap(mapId, spawnId?)`:
   - 清理旧场景对象
   - 创建tilemap（从Tiled JSON）
   - **Image Layer渲染**：如果ground是imagelayer类型，用 `this.add.image()` 铺底图
   - 处理对象层：
     - `furniture_objects` → 放置家具sprite，设置Y排序depth
     - `collision` → 创建碰撞体
     - `npcs` → 创建NPC占位+注册可交互
     - `player_spawn` → 定位出生点
     - `triggers` → 注册触发区（door/dialogue/interact/auto）
     - `items` → 注册可交互物品
   - 创建玩家sprite，设置物理+碰撞
   - 设置世界边界+相机跟随（lerp动态：大地图0.05，小场景0.08）
   - 初始化PlayerController / InteractionSystem / TriggerSystem
4. `listenBridge()` — 监听GameBridge指令（CHANGE_MAP / FREEZE / UNFREEZE）
5. `transitionToMap(mapId, spawnId)` — fadeOut→loadMap→fadeIn
6. `update()` — 驱动各子系统 + Y排序

**TriggerSystem关键逻辑**：
- 每帧检测玩家是否进入trigger zone
- triggerType="walk" → 自动触发（走进通道即切换场景）
- triggerType="interact" → 按E键触发
- triggerType="auto" → 首次进入触发，仅一次
- type="door"的trigger → 通过gameBridge.sendToReact发送TRIGGER_DOOR事件
- React侧接收TRIGGER_DOOR → dispatch CHANGE_MAP → gameBridge.sendToPhaser(CHANGE_MAP) → Phaser侧执行transitionToMap

## MapScene适配Image Layer的关键改动

文档中的MapScene.ts基于tile系统，但本项目全部使用Image Layer。需要关键改动：

### PreloadScene改动
```typescript
// 原方案（tile系统）：
this.load.tilemapTiledJSON(map.mapKey, map.mapJson);
this.load.image(map.tilesetKey, map.tilesetImage);

// 新方案（Image Layer）：
this.load.tilemapTiledJSON(map.mapKey, map.mapJson);
this.load.image(map.groundKey, map.groundImage);  // 地面底图
this.load.image(map.tilesetKey, map.tilesetPath);  // 最小占位tileset
```

### MapScene改动
```typescript
// 原方案：用tile图层渲染
const groundLayer = this.map.createLayer("ground", tileset, 0, 0);

// 新方案：检测ground图层类型
const groundLayerData = this.map.getLayer("ground");
if (groundLayerData && groundLayerData.type === "imagelayer") {
  // Image Layer: Phaser会自动从Tiled JSON中读取image路径并渲染
  // 但需要确保图片已预加载
  this.add.image(0, 0, entry.groundKey).setOrigin(0, 0);
}
```

实际上，Phaser 3 的 `this.make.tilemap()` 会自动处理 Image Layer——它会创建一个 Image 类型的 GameObject。但前提是图片路径和加载key匹配。需要在PreloadScene中用正确的key加载底图。

**最可靠方案**：不用tilemap加载Image Layer，而是在MapScene的loadMap中手动铺底图：
```typescript
// 手动铺底图（最可靠）
const entry = MapRegistry[mapId];
this.add.image(0, 0, entry.groundKey).setOrigin(0, 0).setDepth(0);

// tilemap仅用于读取对象层
this.map = this.make.tilemap({ key: entry.mapKey });
const tileset = this.map.addTilesetImage(entry.tilesetNameInTiled, entry.tilesetKey);
// 不创建任何tile图层，只用tilemap读取对象层数据
```

// DialogOverlay.tsx & CgOverlay.tsx
const PROTAGONIST_NAMES = ["叶平生"];
function getSpeakerLabel(speaker: string | undefined): string | null {
  if (!speaker || NARRATOR_NAMES.includes(speaker)) return null;
  if (PROTAGONIST_NAMES.includes(speaker)) return "我";
  return speaker;
}
