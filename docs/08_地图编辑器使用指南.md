# 地图编辑器使用指南 v1.0

> 工具文件：`scripts/map_editor.html`
> 目标：在浏览器中直接完成地图的碰撞框、多边形遮挡、交互触发器和出生点的可视化编辑，导出标准 Tiled JSON 格式的 `map.json`
> 编写日期：2026-06-07
> 最后更新：2026-06-07

---

## 0. 快速开始

1. 双击或浏览器打开 `scripts/map_editor.html`
2. 点击 **📂 加载底图**，选择场景底图（PNG/JPG/WebP）
3. 选择编辑模式，在底图上绘制
4. 点击 **💾 导出 map.json**，放到 `client/public/assets/maps/<地图名>/map.json`

无需安装任何软件，无需 Node.js，一个 HTML 文件搞定。

## 2. 四种编辑模式

### 2.1 🔲 碰撞框模式（Collision）

**用途**：标记不可通行的墙壁、障碍物等。运行时玩家无法穿越。

**操作**：
- 在底图上**按住左键拖拽**绘制矩形
- 松开鼠标完成绘制
- 点击已有碰撞框可**选中**，选中后可拖拽移动

**导出位置**：`map.json` → `collision` 对象层

### 2.2 🔶 多边形遮挡模式（Polygon Occlusion）

**用途**：用多边形精准勾勒不规则物品的遮挡形状。适用于斜放家具、异形装饰等无法用矩形精确描述的场景。编辑器中将多边形内的底图内容裁剪复制出来，并支持叠加精灵图纹理，实现精准的视觉遮挡效果。

**操作**：
- 在底图上**左键点击**依次添加顶点
- 顶点之间自动连线（虚线），显示多边形轮廓
- 按 **Ctrl+Z** 撤销上一步顶点（可连续撤销，退回到上一步）
- 添加 ≥3 个顶点后，**右键或按 Enter** 闭合多边形
- 按 **Esc** 取消当前绘制

**编辑已有多边形**：
- 点击多边形选中（边框变为橙色实线）
- 选中后顶点显示橙色小方块手柄，**拖拽顶点**可微调形状
- 选中后可拖拽多边形整体移动

**渲染效果**：
- 多边形内部显示从底图裁剪的对应区域（而非纯色填充），视觉上像是"把底图框选的部分提起"
- 可叠加精灵图纹理，精灵图在裁剪底图之上显示

**属性**（选中后在右侧面板设置）：

| 属性 | 说明 |
|------|------|
| 可穿越 | 勾选后不产生碰撞（walkable=true）|
| 精灵图 | 可选关联精灵图，叠加在裁剪底图之上显示 |

**导出位置**：`map.json` → `furniture_objects` 对象层（含 `polygon` 字段）

> **注意**：导入已有的精灵图叠层矩形（如 dormitory 的 item_01~14）时，会自动转为多边形对象。你可以在此基础上继续编辑顶点。

### 2.3 🟢 交互触发模式（Trigger）

**用途**：标记可交互区域（E键触发、走入触发、门切换场景等）。

**操作**：
- 在底图上**按住左键拖拽**绘制矩形
- 选中后在右侧面板设置触发类型

**触发类型**：

| 类型 | 行为 | 使用场景 |
|------|------|---------|
| `interact` | 玩家走到附近按 E 键触发 | 观察物品、对话NPC |
| `walk` | 玩家走入区域自动触发 | 进入房间触发剧情 |
| `auto` | 进入地图即自动触发 | 开场剧情 |
| `door` | 走入后切换地图（有碰撞） | 门/通道 |

**属性**：
| 属性 | 说明 |
|------|------|
| 触发类型 | 上述四种之一 |
| 场景ID | 触发后跳转的剧情节点 sceneId |
| 仅一次 | 勾选后只触发一次 |

**导出位置**：`map.json` → `triggers` 对象层

### 2.4 🚶 出生点模式（Spawn）

**用途**：标记玩家进入地图时的起始位置。

**操作**：
- 在底图上**点击**放置出生点
- 选中后在右侧面板设置出生点ID

**属性**：
| 属性 | 说明 |
|------|------|
| 出生点ID | 如 `spawn_livingroom_door`，地图切换时通过此ID定位 |

**导出位置**：`map.json` → `player_spawn` 对象层

## 3. 通用操作

### 3.1 选中与编辑

| 操作 | 效果 |
|------|------|
| 点击对象 | 选中（边框高亮，右侧显示属性面板）|
| 点击空白处 | 取消选中 |
| 拖拽选中对象 | 移动位置 |
| 在右侧面板修改属性 | 实时更新 |
| Delete / Backspace | 删除选中对象 |

### 3.2 键盘微调

选中对象后：

| 按键 | 效果 |
|------|------|
| ↑↓←→ | 移动 1px |
| Shift + ↑↓←→ | 移动 10px |
| Delete / Backspace | 删除选中对象 |
| Esc | 取消多边形绘制 / 取消选中 |
| Ctrl+Z | 多边形绘制中撤销上一步顶点 |

### 3.3 画布操作

| 操作 | 效果 |
|------|------|
| 滚轮 | 缩放（5% ~ 800%） |
| 中键拖拽 | 平移画布 |
| Alt + 左键拖拽 | 平移画布（替代方案） |

### 3.4 拖放文件

支持将文件直接拖入浏览器窗口：
- **图片文件**：自动作为底图加载（第一个）或精灵图加载（后续）
- **JSON 文件**：自动作为 map.json 导入

## 4. 加载精灵图

精灵图可为碰撞框、多边形遮挡、交互触发等任意对象叠加纹理：

1. 点击 **📥 加载精灵图**，选择 PNG 文件（可多选）
2. 选中任意对象，在属性面板的"精灵图"下拉中选择对应图片
3. 多边形遮挡：精灵图叠加在裁剪的底图之上显示
4. 矩形对象（碰撞/触发）：精灵图直接绘制在对象矩形区域
5. 导出时会在对应对象中添加 `spriteKey` 属性

> **提示**：精灵图适用于需要纹理显示的矩形物品（如 dormitory 的家具）。对于多边形遮挡，精灵图提供额外的纹理层。

## 5. 导出与导入

### 5.1 导出 map.json

点击 **💾 导出 map.json**，浏览器自动下载 JSON 文件。

导出前确认：
- 地图名（右侧面板顶部输入框，默认 `new_map`）
- 底图文件名（右侧面板顶部输入框，默认 `ground.png`）

导出的 JSON 结构：

```json
{
  "tiledversion": "1.10.2",
  "type": "map",
  "orientation": "orthogonal",
  "width": 44, "height": 24,
  "tilewidth": 32, "tileheight": 32,
  "layers": [
    { "name": "ground",           "type": "imagelayer",   "image": "底图.png" },
    { "name": "collision",        "type": "objectgroup",  "objects": [...] },
    { "name": "furniture_objects", "type": "objectgroup", "objects": [...] },
    { "name": "triggers",         "type": "objectgroup",  "objects": [...] },
    { "name": "player_spawn",     "type": "objectgroup",  "objects": [...] },
    { "name": "npcs",             "type": "objectgroup",  "objects": [] },
    { "name": "items",            "type": "objectgroup",  "objects": [] }
  ]
}
```

多边形对象示例：

```json
{
  "name": "sofa_oblique",
  "type": "furniture",
  "x": 400, "y": 300,
  "width": 0, "height": 0,
  "polygon": [
    { "x": 0,   "y": 0   },
    { "x": 120, "y": 10  },
    { "x": 115, "y": 80  },
    { "x": 5,   "y": 85  }
  ],
  "properties": [
    { "name": "walkable", "value": false, "type": "bool" }
  ]
}
```

### 5.2 导入 map.json

点击 **📥 导入 map.json**，选择已有的 JSON 文件。

导入规则：
- `collision` 层 → 碰撞框对象
- `furniture_objects` 层：
  - 含 `polygon` 字段 → 多边形遮挡对象
  - 不含 `polygon` 字段（矩形）→ 自动转为多边形（4顶点矩形）
- `triggers` 层 → 交互触发对象
- `player_spawn` 层 → 出生点对象

> **注意**：导入不会自动加载底图。如果底图未加载，需要手动加载。

## 6. 完整工作流程示例

以创建一个新地图为例：

### Step 1：准备素材

- 场景底图（PNG，如 `客厅.png`）
- 可选：物品精灵图（PNG，256×256 透明裁切）

### Step 2：打开编辑器

浏览器打开 `scripts/map_editor.html`

### Step 3：加载底图

点击 📂 加载底图 → 选择 `客厅.png` → 自动适应窗口

### Step 4：绘制碰撞框

切换到 🔲 碰撞框模式 → 沿墙壁拖拽绘制矩形

### Step 5：绘制多边形遮挡

切换到 🔶 多边形遮挡模式 → 点击添加顶点勾勒不规则物品轮廓 → 右键闭合

### Step 6：放置交互触发器

切换到 🟢 交互触发模式 → 拖拽绘制 → 设置触发类型和场景ID

### Step 7：放置出生点

切换到 🚶 出生点模式 → 点击放置 → 设置出生点ID

### Step 8：导出

设置地图名（如 `livingroom`）和底图文件名（`客厅.png`）→ 点击 💾 导出 map.json

### Step 9：部署

将导出的 JSON 文件放到：

```
client/public/assets/maps/livingroom/map.json
```

确保底图文件也在同一目录下：

```
client/public/assets/maps/livingroom/客厅.png
```

### Step 10：注册地图

在 `client/src/game/config/mapRegistry.ts` 中注册新地图（参考 `07_地图系统开发文档_v1.0.md` 第 8.2 节）。

## 7. 与 Tiled 编辑器的对比

| 特性 | map_editor.html | Tiled Map Editor |
|------|:---:|:---:|
| 无需安装 | ✅ | ❌ 需下载安装 |
| 多边形绘制 | ✅ 点击添加顶点 | ✅ 多边形工具 |
| 矩形碰撞 | ✅ 拖拽 | ✅ 拖拽 |
| 触发属性面板 | ✅ 内建 | ⚠️ 需手动配置 |
| 精灵图关联 | ✅ 所有对象类型 | ❌ |
| 导入已有地图 | ✅ | ✅ |
| 导出 Tiled JSON | ✅ | ✅ |
| 底图裁剪填充 | ✅ 多边形内显示底图 | ❌ |
| 精灵图叠加预览 | ✅ 实时 | ❌ |
| 顶点编辑 | ✅ 拖拽 | ✅ |
| 多边形撤销 | ✅ Ctrl+Z 撤销顶点 | ✅ |
| Tile 图层编辑 | ❌ | ✅ |

> **建议**：本项目的 Image Layer 方案不需要 tile 编辑，`map_editor.html` 完全满足需求。仅在需要 tile 图层时考虑使用 Tiled。

## 8. 注意事项

1. **多边形至少 3 个顶点**才能闭合，运行时需 ≥3 个顶点才会渲染
2. **导出前确认底图文件名**与 `mapRegistry` 中引用的路径一致
3. **导入已有 map.json 时**，矩形家具对象会自动转为多边形，可继续编辑顶点
4. **多边形遮挡**在编辑器中显示从底图裁剪的内容，并可叠加精灵图纹理
5. **触发器的 sceneId** 需要与 `client/src/data/scenes.ts` 中定义的剧情节点 ID 对应
6. **出生点至少放置一个**，否则玩家会在地图中心生成
7. 文件完全在浏览器端运行，数据不会上传到任何服务器
