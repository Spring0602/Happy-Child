# 开发路线图

## Demo 定位（已确认）

- **Demo内容**：第1-8章，主角叶平生与内在"我"达成和解的认知弧。
- **评判维度**：采用完整哲学框架（认知→行动→影响三层），不因Demo只做认知弧而简化。
- **长期目标**：比赛结束后继续完成整个副本（当前为1/3），作为个人作品集。
- **核心创新**：玩家选择→AI综合分析七维人格画像→动态匹配最符合"事物自然规律"的结局。

## 第一阶段：最小可运行原型

目标：让游戏能在浏览器中跑起来。

- [ ] 前端能显示剧情文本。
- [ ] 点击“继续”能进入下一段剧情。
- [ ] 点击选项能跳转剧情。
- [ ] 玩家人格值能变化。
- [ ] 状态面板能显示人格值。
- [ ] 本地存档能保存和读取。

## 第二阶段：剧情扩展（认知弧）

目标：把第1-8章改编成可玩节点，聚焦"和解"认知弧。

**核心场景序列**（Demo闭环）：
1. `prologue_email`：收到邮件，展示玩家风险判断。
2. `home_planbook`：家庭规则，理解"快乐"被强制。
3. `school_exam`：学校初入，优秀学生品质测试。
4. `liuyu_meet`：刘宇登场，测试玩家是否值得合作。
5. `art_class`：美术课，王老师提示"做自己"。
6. `wang_trade`：王老师交易，三个谜题。
7. `ritual_gong`：功成名就祈愿仪式（核心展示场景）。
8. `rooftop_dialogue`：天台与"我"的对话（Demo核心选择点）。
9. `ending_judge`：AI裁决，展示人格画像与结局。

**评判维度**：采用完整七维人格框架（认知→行动→影响三层），不因Demo只做认知弧而简化。

- [ ] 序章：收到邮件。
- [ ] 家庭区域：计划表、食物、镜子规则。
- [ ] 学校区域：考试、刘宇、空座位。
- [ ] 美术课：王老师“做自己”。
- [ ] 王老师交易：三个谜题。
- [ ] 3班逃生。
- [ ] 全校追杀。
- [ ] 删除“好孩子”。
- [ ] 天台对话。

## 第三阶段：AI 模块（三层评判框架）

目标：让 AI 成为玩法机制，采用完整"认知→行动→影响"三层评判框架。

- [ ] 选择后 AI 分析玩家人格（七维画像）。
- [ ] NPC 根据玩家画像释放不同等级线索。
- [ ] 结局由 AI 在预设结局池中裁决（认知层→行动层→影响层）。
- [ ] AI 输出不能破坏世界观。
- [ ] **核心创新展示**：同一场景（天台和解）→ 不同玩家画像 → 2-3种不同和解方式。

## 第四阶段：规则系统增强（Demo可选）

目标：让副本更像游戏；Demo可跳过此阶段，专注认知弧。

- [ ] 加入探索度。
- [ ] 加入叛逆值。
- [ ] 加入违规压力。
- [ ] 加入 NPC 信任度。
- [ ] 加入快乐证明度。

**注意**：Demo聚焦"和解"认知弧，规则系统可在比赛后继续完善。

## 第五阶段：表现层增强（比赛重点）

目标：提升比赛观感；Demo聚焦认知弧，表现层需精致。

- [ ] 完善各场景像素地图（livingroom/bedroom/bathroom/classroom等）。
- [ ] 扩充角色立绘表情集（yps多表情、刘宇/王老师/周骐瑞立绘）。
- [ ] 加入规则警告动效（违规压力UI）。
- [ ] 加入镜中空间滤镜（家庭线核心视觉）。
- [ ] 加入音效和背景音乐（氛围感）。
- [ ] 制作 Demo 演示视频（3分钟以内，展示核心创新点）。

**注意**：Demo 不需要开放世界，但需要风格统一、氛围到位。场景背景已改用 Phaser 像素地图（不再使用 `bg/` 中的 SVG 背景图）。

## 第六阶段：部署与提交（比赛要求）

目标：满足比赛提交要求；Demo聚焦"和解"认知弧，评判维度采用完整哲学框架。

- [ ] 前端构建成功（Vite + React + TypeScript）。
- [ ] 后端 AI 服务可用（Express + TypeScript + 真实LLM API）。
- [ ] 在线体验链接可访问（比赛评委可直接体验）。
- [ ] 准备作品介绍 PPT（核心创新：AI综合分析玩家选择，动态裁决结局）。
- [ ] 导出 CodeBuddy 历史对话记录（开发过程文档）。
- [ ] 准备 3 分钟以内 Demo 视频（展示：收到邮件→天台和解→AI裁决结局）。

**Demo 核心展示点**：
1. 玩家选择→AI综合分析七维人格画像→动态匹配结局（不是"选项→线路→结局"）。
2. 同一场景（天台和解）→不同玩家画像→2-3种不同和解方式。
3. 评判维度采用完整哲学框架（认知→行动→影响三层），不因Demo只做认知弧而简化。
