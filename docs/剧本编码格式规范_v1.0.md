# 剧本编码格式规范 v1.0

> 目的：在你做叙事决策后，我能零歧义地翻译成代码。
> 使用方式：每段剧情用一个 `###` 标题块，按约定标签描述，我自动生成 scenes.ts 配置 + MapScene 逻辑。

---

## 一、角色归属标记（解决"谁说的"歧义）

在对话行前用 `[角色码]` 标记，四选一：

| 角色码 | 含义 | 在对话框中显示 |
|--------|------|----------------|
| `[旁白]` | 第三人称叙述 | **仅显示正文**，不显示任何 speaker 名称 |
| `[主角]` | 叶平生内心独白 | speaker 显示 **"我"** |
| `[主角说]` | 叶平生说出声的话语 | speaker 显示 **"我"**，可能触发 NPC 响应 |
| `[NPC:名字]` | 指定 NPC 说的话语 | speaker 显示 **NPC 名字**；有立绘时右下角显示立绘 |

**示例：**
```
[旁白] 雨水打在阳台的铁皮棚上，发出细密的声响。
[主角] 我到底在怕什么？
[主角说] 这雨真大啊。
[NPC:陈煜浩] 平生，你怎么还不睡？
```

### NPC 立绘规则

- 每个 NPC 在 `剧本全局设定` 中声明是否拥有立绘及立绘路径
- `[NPC:名字]` 说话时，有立绘者自动在**屏幕右下角**显示对应立绘，无立绘者只显示名字
- 立绘随对话推进自动切换为当前说话 NPC 的立绘

> **关键**：`[主角]` 和 `[主角说]` 的区别在于——内心独白不影响 NPC 状态，说出声的话可能被 AI/NPC 响应。

## 二、模式切换标记（解决"CG还是探索"的歧义）

每个 `###` 块的第一行用模式标记：

```
### [CG] 场景名
### [地图] 场景名
### [混合] 场景名
```

### 2.1 `[CG]` 模式 —— 全屏大图 + 底部对话

```
### [CG] 第一场：完成项目
图片：/assets/CG/前兆/idea界面.png
[旁白] 终于解决完所有的bug了...
[旁白] 现在是04:46。我，叶平生...
[主角] 好累啊，真想到阳台去透透气。
→ 进入宿舍地图 dormitory spawn_sit_desk
```

**关键：** 没有 `[玩家]` 交互，纯阅读推进。`→` 后面的"跳转到地图"就是结束点。

### 2.2 `[地图]` 模式 —— RPG 自由探索 + 对话叠层

```
### [地图] 第三场：探索宿舍
地图：dormitory
出生点：spawn_sit_desk
冻结玩家：是                ← 是否冻结（默认"否"）
NPC：
  - id: npc_chen, 精灵: chen_haoframes, 位置: spawn_spawn_53
  - id: npc_a, 精灵: yps_frames, 位置: spawn_spawn_54

[玩家] 自由移动:空格键推进对话
[旁白] 终于完成了项目，该休息了。
```

### 2.3 `[混合]` 模式 —— 地图上自由探索，触发 interactable 后进入对话

```
### [混合] 第四场：自由探索中的交互
地图：dormitory
出生点：spawn_stand_chair_right

--- 可交互点 ---
@trigger_pc
  提示文字：E查看电脑
  触发：[CG] 查看电脑CG
    图片：/assets/CG/前兆/idea界面.png
    [旁白] 光标停留在5154行的位置...
    → 回到地图 spawn_stand_chair_right

@trigger_clock
  提示文字：E时钟
  触发：[对话]
    [旁白] 04:46。

@trigger_window
  提示文字：E前往阳台
  条件：!dorm_act2_flag          ← flag 条件：未触发时才有效
  触发：[对话→跳转]
    [旁白] 我轻脚走到阳台...
    → 切换到阳台地图 balcony_night spawn_spawn_77
```

> **重要**：`@trigger_xxx` 名称必须对应 map.json 中的 `id` 字段。条件用 `!` 前缀表示"flag 为 false 时才触发"，无前缀表示"flag 为 true 时触发"。

## 三、NPC 行为标记

### 3.1 NPC 全局设定（写在剧本文件最开头）

```
@NPC设定
  刘宇:     立绘: ly_smile,     表情: [默认为default, 还可用smile/sad/angry/surprised等]
  叶平生:   立绘: yps_defult,   表情: [主角立绘，一般用于CG]
  陈煜浩:   立绘: 无,           精灵: chen_haoframes
  室友A:    立绘: 无,           精灵: yps_frames
  ...
```

> **立绘命名规则**：`/assets/portraits/{首字母}_{表情}.png`
> 如刘宇微笑 → `ly_smile.png`，刘宇默认 → `ly_default.png`
> `立绘: 无` 的 NPC 说话时只显示名字，不出现立绘。

### NPC 表情切换

在对话中可以用 `表情:` 后缀切换当前 NPC 的立绘表情：

```
[NPC:刘宇] 表情: smile
[NPC:刘宇] 今天天气真好啊。
[NPC:刘宇] 表情: angry
[NPC:刘宇] 但你这事做得太过分了！
```

> ⚠️ **容错规则**：实际加载时检查 `/assets/portraits/{首字母}_{表情}.png` 是否存在。
> - 文件存在 → 显示立绘
> - 文件不存在（该表情还没画） → 不显示立绘，只显示 NPC 名字，**不报错**
> 
> 这样你可以先写完整剧本标注表情，目前只有一张立绘也没关系——表情还缺的对话自动退化为纯文字。

> 不指定 `表情:` 时默认使用 `default`。`[旁白]` 和 `[主角]` 说话时会清除 NPC 立绘。

> `立绘: 无` 的 NPC 说话时只显示名字，不出现立绘。

### 3.2 NPC 出场定义（在 `[地图]` 块开头）

```
NPC：
  - id: npc_chen, 名字: 陈煜浩, 位置: spawn_spawn_53, 朝向: 下
  - id: npc_b, 名字: 室友A, 位置: spawn_spawn_55, 朝向: 左
```

> `名字` 必须匹配 `@NPC设定` 中的名字，立绘和精灵帧自动关联。

### 3.3 NPC 动作指令（在对话序列中）

```
NPC动作：
  npc_chen → 播放动画: idle_down → 等待2秒 → 播放动画: walk_left → 移动至: spawn_spawn_60
  npc_b → 面朝: 右
```

## 四、特效与音效标记

在 `###` 块的开头或对话序列中标注：

```
### [地图] 阳台场景
地图：balcony_night
出生点：spawn_spawn_77
效果：rain_start                  ← 进入时开启下雨
音效：rain_ambient (loop)         ← 背景雨声循环
```

对话中可以切换效果：

```
[旁白] 雨渐渐小了...
→ 效果: rain_stop
→ 音效: stop rain_ambient
```

支持的指令：

| 指令 | 含义 |
|------|------|
| `→ 效果: rain_start` | 开启下雨粒子 |
| `→ 效果: rain_stop` | 停止下雨 |
| `→ 效果: flash_screen` | 屏幕闪烁白光 |
| `→ 效果: fade_out` | 渐暗 |
| `→ 效果: fade_in` | 渐亮 |
| `→ 音效: xxx (loop)` | 播放循环音效 |
| `→ 音效: stop xxx` | 停止音效 |
| `→ 音效: xxx (once)` | 播放一次性音效 |
| `→ BGM: xxx (loop)` | 切换背景音乐 |

## 五、选项分支标记

> ⚠️ **人格画像由 AI 动态裁定**，不硬编码特质加减分。
> 你只需标注选项的**语义标签**，AI 在运行时根据玩家完整行为轨迹判定各维度变化。

```
[主角] 这么晚了，该睡觉了吧？
→ 选项：继续探索
    AI标签：反叛, 拖延, 逃避约束
    剧情：继续自由探索
    设置flag: dorm_act2_exploring

→ 选项：睡觉
    AI标签：服从, 自我约束
    剧情：进入睡觉片段
    跳转：dorm_act2_sleep_result
```

> `AI标签:` 用逗号分隔，描述这个选择的行为倾向，AI 根据标签 + 上下文裁决七维画像变化。
> 如果选择被重要 NPC 当场看到，加一行：
> ```
> NPC目睹：刘宇, 王老师    ← AI 以这些 NPC 的视角动态评估，影响后续信任/态度
> ```

## 六、AI NPC 模式 · 角色设定

### 6.1 @角色设定（写在剧本最开头，与 @NPC设定 并列）

重要 NPC 需要角色设定卡片，供 AI 扮演和动态判定使用：

```
@角色设定 刘宇
  身份：班长 / 表面朋友 / 危险合作者
  性格：外向聪明讲义气，用玩笑掩盖真实意图
  初始态度：观察为主，不完全信任主角
  AI权限：根据主角行为决定是否释放学校区域线索
  禁止行为：
    - 不能直接说出学校根本规则
    - 不能直接告诉玩家删除"好孩子"规则
    - 不能承认自己受系统控制
```

> 当前已完成角色设定：刘宇、王沁林(王老师)、周隽秀
> **待补写设定**：林芷萱、周骐瑞、？？？ — 首次出场前写好即可，后续我会提醒。

### 6.2 角色观念的动态变化

NPC 经历事情后观念会改变，游戏运行时由 AI 维护。你需要在关键转折点标记：

```
[NPC:刘宇] 表情: serious
[NPC:刘宇] 你居然真的帮了周隽秀...我没想到。
→ NPC观念更新: 刘宇                  ← 标记此事件应影响刘宇对主角的态度
```

> 不写 `→ NPC观念更新` 时，每次 `NPC目睹` 的选项都会自动触发 AI 评估。

### 6.3 整段对话由 AI 扮演 NPC（半 AI 模式）

不适合写死的过渡性对话（试探、闲聊、态度检验）交给 AI 生成：

```
### [AI对话] 刘宇的试探
背景：走廊，放学后，只有两人
NPC自主扮演: 刘宇
AI提示：刘宇注意到主角今天在教室里的表现，想试探主角是否察觉了什么。
        如果主角表现出合作信号，透露一点学校规则线索。
        如果主角不信任，打哈哈转移话题。
→ 对话结束后：回到地图 corridor
```

> `[AI对话]` 模式适合 NPC 的试探/闲聊/态度检验，**关键剧情仍需写死**。

### 6.4 帮助分水岭

```
[NPC:刘宇] ...行吧，你跟我来。
→ NPC给予帮助: 刘宇 → 解锁区域: classroom_3
→ 跳转：ch5_liuyu_guide
```

## 七、flag 条件与操作

### 6.1 条件触发

```
@trigger_pc
  条件：dorm_act2_exploring       ← 有这个 flag 才触发的版本
  提示文字：E关电脑
  触发：[对话]
    [旁白] 身体已经非常疲惫了，你爬上床。
    → 设置flag: dorm_sleeping
    → 跳转：dorm_act2_sleep_result
```

### 6.2 Flag 操作

| 写法 | 含义 |
|------|------|
| `→ 设置flag: xxx` | 设为 true |
| `→ 清除flag: xxx` | 设为 false |
| `条件：flag_name` | flag == true 才执行 |
| `条件：!flag_name` | flag == false 才执行 |

## 九、格式摘要速查

```
### [模式] 场次名
图片/地图：路径                              ← CG模式用"图片"，地图模式用"地图"
出生点：spawn_id                              ← 地图模式
冻结玩家：是/否                               ← 默认否
效果：rain_start / rain_stop / flash_screen
音效：name (loop/once) / stop name
BGM：name (loop)
NPC：
  - id: xxx, 名字: xxx, 位置: spawn_xxx, 朝向: 上下左右

// 对话
[角色码] 对话内容...

// 选项
→ 选项：选项文本
    AI标签：标签1, 标签2
    NPC目睹：角色名（可选）
    剧情：一句话描述
    跳转：sceneId
    设置flag: xxx

// 交互点（地图模式）
@trigger_id
  条件：flag_name 或 !flag_name
  提示文字：E按钮文字
  触发：[CG] 或 [对话]
    [角色码] 内容...
    → 回到地图 / 切换地图 / 跳转 / 设置flag / 效果 / 音效

// NPC动作
NPC动作：
  npc_id → 播放动画: anim_name → 移动至: spawn_xxx → 面朝: 方向

// 流程控制
→ 跳转：sceneId
→ 切换地图: map_id
→ 出生点: spawn_id
→ 回到地图 spawn_id
→ 设置flag: xxx
→ 清除flag: xxx
→ 冻结玩家：是/否
→ 效果: effect_name
→ 音效: sound_name (loop/once)
→ BGM: bgm_name (loop)
→ 播放NPC动画: npc_id anim_name
→ 按空格键（等待玩家输入）
→ 等待: N秒（自动延迟N秒）
→ 画面: fade_out / fade_in
→ NPC目睹：角色名（选项旁）
→ NPC观念更新: 角色名
→ NPC给予帮助: 角色名 → 解锁区域: xxx
```

## 十、快速参考：常见模式

| 你想要的叙事效果 | 对应的格式片段 |
|------------------|---------------|
| 全屏大图+底部旁白纯阅读 | `### [CG] ... [旁白] 文本 → 跳转/onCgEnd` |
| 地图上自由走，靠近某物体按E | `### [混合] ... @trigger_xxx 提示文字：Exxx 触发：[对话]` |
| 进入新地图自动弹出对话 | `### [地图] ... 冻结玩家：是 [旁白] 文本 → 冻结玩家：否` |
| 对话中选择分支 | `→ 选项：文本 AI标签：... 跳转：sceneId` |
| NPC 在对话中做动作 | `NPC动作： npc_id → 播放动画: walk → 移动至: spawn_xxx` |
| 按条件切换同一交互点行为 | `@trigger_pc 条件：!flag → [对话] A` |
| 自动播放序列（无玩家输入） | `→ 等待:2秒 → [旁白] 文本 → 等待:1.5秒 → [主角] 文本` |
| NPC 试探主角，AI 生成对话 | `### [AI对话] ... NPC自主扮演: 角色名 AI提示：...` |
| 选项被 NPC 看到并评价 | `→ 选项：... NPC目睹：刘宇` |
| NPC 态度转折，给予帮助 | `→ NPC给予帮助: 刘宇 → 解锁区域: xxx` |
| NPC 经历重要事件，观念更新 | `→ NPC观念更新: 刘宇` |

---

> **最后说明**：这个格式的核心原则是——**你每行写的内容，都能直接对应到一行代码或一组特定的事件调用**。你不需写代码，只需用这些标签"声明"你想要什么效果，我来填实现细节。
>
> 如果某个效果这个格式覆盖不了，我们可以在那个具体场景上临时约定新标签——这样格式本身也会不断进化。
